# Системная документация

Этот раздел отделяет устройство системы от продуктовых документов и от
визуального дизайна.

- [`architecture.md`](architecture.md) — общая архитектура Uprava: границы
  Core, Node Daemon и клиентов, владение состоянием и основные контракты.
- [`areas/`](areas/) — отдельные системные направления с собственной vision,
  архитектурой, lifecycle, контрактами и открытыми вопросами.

Общие решения, затрагивающие несколько направлений, фиксируются в
`architecture.md`. Подробная проработка одного механизма остаётся в
соответствующем документе `areas/`. Когда направление приводит к изменению
продуктового scope или порядка реализации, обновляются также документы верхнего
уровня в `docs/`.
