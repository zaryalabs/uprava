## M7 closure update (2026-07-11)

M7 is complete. Known command and event payloads are tagged typed variants;
only their explicit `extension` variants accept extension data. Core and Node
reject kind/payload mismatches before durable effects. Command targets are
tagged identities, capability values and action capabilities are typed, and
the session UI consumes Core's `available_commands` projection instead of
reimplementing runtime transitions.

`protocol-check` now covers 18 Web-facing enum roots and Rust-generated
canonical fixtures for every runtime-validated HTTP, SSE and terminal root.
The closure gate passed with Node 89, Protocol 11 and Core 134 plus binary 1;
the focused Web regressions pass while the final aggregate rerun follows the
M8 work.

  `reference`, `domain`, `control`, `workspace`, `api`, `projection` and

  and Web-facing Rust callers do not need path changes.

  enum literal drift remains checked after the split.
- Replaced the optional Node/Placement/session/runtime command identity bag
  with the tagged `CommandTarget` variants `node`, `placement` and
  `session_runtime`; legacy top-level target fields are no longer emitted.
- Replaced every known Core-to-Node command payload with a tagged
  `CommandPayload` variant. `CommandKind::Extension` is the only extension
  escape hatch, and both Core and Node reject kind/payload mismatches before
  persistence or execution.
- Added the typed `ActionCapability` wire enum and made the Core session
  projection publish legal actions through it instead of arbitrary strings.
- Replaced arbitrary known capability values with tagged `CapabilityValue`
  variants and an explicit `extension` escape hatch.
- Added canonical Rust-generated Web fixtures to `protocol-check`; Vitest runs
  those fixtures through the high-risk Zod validators.
- Added a versioned, bounded canonical reference key for inspector URLs;
  arbitrary percent-encoded JSON and malformed/unversioned keys are rejected.
- Added route-level lazy loading and a production manifest budget. The current
  initial graph is 121.2 KiB gzip, excludes Monaco/xterm, and the workspace
  route is emitted separately.
- Added Placement/path-keyed workspace drafts with base/local/dirty/conflict
  state, remote conflict preservation, discard/reload controls and an unload
  warning.
