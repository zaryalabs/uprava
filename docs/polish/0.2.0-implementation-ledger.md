# Uprava 0.2.0 — implementation ledger

Status: active historical evidence

Updated: 2026-07-10

Branch baseline: `2e4565e`

This document preserves the useful rationale and residual state from the
ignored `.local/context/0.2.0-execution.md` journal. It is not a second work
plan. Future ordering belongs to the
[completion plan](../tmp-plans/0.2.0-completion-from-current.md), and final
scope belongs to the
[quality-foundation plan](../tmp-plans/0.2.0-quality-foundation.md).

Git history is the source of truth for exact diffs. This ledger groups the
important implementation commits by boundary and records why the boundary is
not necessarily complete.

## Baseline and execution-mode history

- Source implementation baseline: `d05621c` (`0.1.8`).
- Audit/plan baseline: `10e486e` (`docs: audit`).
- Current implementation baseline: `2e4565e`.
- The original execution journal used fine-grained execution units and
  worker/reviewer/manager handoffs.

## Protocol, auth and initial observability

Commits:

- `c2feb06` — centralized protocol compatibility helper.
- `28c82e5` — coordinated protocol switch to `v2`; retained v1 is rejected.
- `189af75` — enrollment claim and security audit committed atomically.
- `d450371` — Argon2id password records; legacy SHA-256 fails closed.
- `5f1e1b9` — initial bounded Core metrics.
- `847bdbc`, `175ea2d` — Node reliability counters.

Evidence:

- protocol compatibility/negative-version tests pass;
- `protocol-check` is part of `make dl`/`make c` and compares Rust enum wire
  literals, tracked Web literal constants and handwritten tagged-object
  discriminants for 16 Web-facing roots;
- Web protocol aliases and runtime validators consume tracked generated
  literals from `apps/web/src/shared/protocol/literals.ts`;
- high-risk Web boundaries now validate async workspace command resources,
  selected command/terminal HTTP responses, session SSE envelopes and terminal
  stream frames;
- password and enrollment-audit tests pass;
- Core metrics exclude high-cardinality labels in the covered counters.

Residual:

- `v2` now has a tracked enum-literal drift gate and selected Web runtime
  validators, but not full JSON Schema/codegen coverage for all Web-facing
  shapes;
- `uprava-protocol` still mixes domain/API/control concerns and needs a module
  split;
- command/event payloads and target bags still need typed variants plus an
  explicit extension escape hatch;
- public-ingress quotas, bounded logging, full metrics, response correlation,
  OTLP and telemetry drop accounting remain.

Milestone 2 evidence:

- bounded generation-aware control connections activate only after valid Hello
  and stale generations cannot remove or reactivate newer routes;
- ACK/result ownership and transitions, typed result echoes, event resource
  ownership and terminal ownership are checked before mutation;
- frame/batch/output limits and queue saturation have observable rejection
  paths;
- cross-Node forgery, stale-generation, conflicting-result and queue tests
  pass; Core reached 128 tests and `make l` passed.

## Verification chronology summary

Repeated milestone checkpoints passed Rust format/Clippy, Rust tests,
TypeScript lint/typecheck, Web tests/build, dependency policy and documentation
checks. Historical test counts changed as regressions were added. The current
portable baseline is the fresh full gate recorded in the
[handoff](0.2.0-handoff.md), not an older intermediate count.

Known historical environment issue:

- the Node enrollment HTTP test uses an ephemeral localhost listener and was
  blocked or flaky under some sandbox executions;
- full and isolated escalated runs have passed it, while a later full repeat
  returned HTTP 503 after the other 73 tests passed;
- treat a future failure as a real gate until the environment and fixture are
  distinguished, rather than silently skipping it.
