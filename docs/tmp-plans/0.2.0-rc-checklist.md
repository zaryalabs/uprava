## Remaining release gates

1. Complete the remaining M7 protocol module split, typed command/event
   payloads, broader schema generation and authoritative action-capability
   projection; then complete M8 Web workbench ownership/lazy-loading.
2. Complete live M6/M9 reconnect and runtime E2E evidence (state-store
   replay/regression is implemented).
3. Repeat basic deterministic checks as needed; final live E2E and clean-state
   reset are intentionally reserved for the user's final review.
4. Enter `0.2.0-rc.1` only after the bulk implementation gates above pass.

## Release rule

Never rebuild the same `0.2.0-rc.N` with different source. Every candidate
records its Git SHA, package versions, Core/Web/Node artifact ids and state-slot
procedure. Final `0.2.0` is cut only after the candidate passes the complete
clean-state suite twice: once for the final RC and once for the immutable final
build.
