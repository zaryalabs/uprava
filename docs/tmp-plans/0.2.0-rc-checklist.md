## Remaining release gates

1. Commit the unique `0.2.0-rc.2` source and record its Git SHA.
2. Build and run all production images plus the complete clean-state,
   backup/restore, rollback, deterministic browser and real Codex smoke suite
   against that immutable SHA.
3. Review the complete branch diff and record artifact ids/manifest.
4. Hand the immutable RC to the user for the separate live E2E gate.

## Release rule

Never rebuild the same `0.2.0-rc.N` with different source. Every candidate
records its Git SHA, package versions, Core/Web/Node artifact ids and state-slot
procedure. Final `0.2.0` is cut only after the candidate passes the complete
clean-state suite twice: once for the final RC and once for the immutable final
build.
