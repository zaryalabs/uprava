## Результат версии

`0.2.6` переносит согласованную workspace-centered information architecture из
статического прототипа в настоящий React Web Control Panel без изменения
Core/Node authority boundary:

```text
Dashboard
Nodes
  Node Overview
    Workspace
      Agent
      Workbench
      Jobs
```

Версия должна изменить способ композиции уже реализованных функций, а не
переписать runtime, workspace или Job mechanics. Agent session, файлы, editor,
diff, PTY terminal и Background Jobs остаются теми же Core-owned объектами и
используют существующие API.

## Проверенный baseline 0.2.5

### Что уже есть и переиспользуется

- `AppShell` уже загружает единый `InventorySnapshot` и показывает дерево
  `Node -> Placement -> Session`.
- `NodeDetailRoute` уже является основой Node Overview: presence, heartbeat,
  capabilities, credentials, revoke/delete и workspace list.
- `PlacementRoute` уже загружает placement, запускает session и монтирует весь
  `WorkspaceInspector`.
- `SessionRoute` уже содержит lifecycle controls, SSE projection, timeline,
  approvals, composer, scheduled messages, evidence и agent projection.
- `WorkspaceInspector` уже содержит file tree, Monaco editor, diff, xterm PTY,
  bounded command runner и command history.
- Jobs уже имеют полный create/list/detail/run flow и всегда хранят
  `project_placement_id`.
- Inspector stack уже кодируется в query parameter `inspect`, умеет хранить
  стек references и закрываться через `Escape`.
- `/health`, `/inventory` и `/jobs` дают все данные, необходимые для четырёх
  Dashboard metrics, workspace filtering и основного Recent Activity.

### Основные разрывы с VDR

| Область | Реализация 0.2.5 | Цель 0.2.6 |
| --- | --- | --- |
| Global navigation | `Dashboard`, `Jobs`, `Nodes` | Только `Dashboard`; Nodes являются деревом в sidebar |
| Sidebar | Node -> Workspace -> Session, всегда открыт | Node -> Workspace, независимо скрывается toggle-кнопкой |
| Workspace entry | Сразу длинный file inspector и кнопка Start | Общий workspace shell с `Agent / Workbench / Jobs` |
| Agent | Отдельный `/sessions/:id` | Session list и выбранная session внутри workspace Agent surface |
| Workbench | Вертикальный набор Files, Terminal, Command, Diff, History | IDE-like grid: file tree, editor/diff и terminal; без Command block |
| Jobs | Глобальный `/jobs` с выбором workspace в create form | Список, create, detail и run history внутри текущего workspace |
| Inspector | Постоянно резервирует `320px`, даже пустой | Закрыт по умолчанию; колонка/drawer появляется только для reference |
| Dashboard | Overview, pipeline, sessions и attention | Четыре metrics и компактный Recent Activity |
| Status | Несколько равнозначных badges | Явные presence/lifecycle/attention/workspace dimensions |

## Граница backend и данных

### Решение для 0.2.6

Изменения Rust protocol, Core routes, SQLite schema и Node Daemon **не входят**
в план и по текущему аудиту не требуются.

- `InventorySnapshot.nodes` даёт presence и `active_runtime_count`.
- `InventorySnapshot.placements` связывает workspace с Node.
- `InventorySnapshot.sessions` связывает session с placement и даёт
  `updated_at`.
- `JobSummary.project_placement_id` позволяет фильтровать Jobs по workspace.
- `JobSummary.latest_run` достаточен для подсчёта running Jobs: invariant 0.2.5
  допускает не более одного active run на Job, а latest run является текущим.
- `HealthResponse` остаётся source of truth для Core API status.

### Честная граница Recent Activity

В `0.2.6` Recent Activity является UI projection из уже доступных read models:

- session updated/state rows из `InventorySnapshot.sessions`;
- latest Job Run rows из `JobSummary.latest_run`;
- сортировка по `updated_at`/`queued_at` и ограничение последними элементами;
- переходы ведут в соответствующий workspace Agent или Jobs surface.

Строки должны описывать наблюдаемое состояние (`Session updated`, `Job run
running`), а не выдумывать точный transition (`became blocked`), которого нет в
read model.

Иллюстративные строки прототипа про точный момент stale heartbeat, workspace
change и check completion не входят в acceptance criteria 0.2.6. Для них нужен
глобальный event/activity projection с устойчивыми timestamps и refs. Это часть
следующего causality/trace slice, а не скрытое расширение UI-релиза. Кнопка
`Open Event Log` не переносится, пока такого route/API нет.

## Канонические маршруты 0.2.6

```text
/dashboard

/nodes/pair
/nodes/:nodeId
/nodes/:nodeId/placements/new

/workspaces/:placementId/agent
/workspaces/:placementId/agent/:sessionThreadId
/workspaces/:placementId/workbench
/workspaces/:placementId/jobs
/workspaces/:placementId/jobs/new
/workspaces/:placementId/jobs/:jobId
/workspaces/:placementId/jobs/:jobId/runs/:jobRunId

/settings/runtime
```

`/workspaces/:placementId` является resolver route. Он открывает последнюю
использованную поверхность этого workspace из local UI preference, а при её
отсутствии — `Agent`.

### Совместимость старых deep links

Стабильные refs и старые bookmarks не должны ломаться:

- `/sessions/:sessionThreadId` загружает/находит session и перенаправляет в
  `/workspaces/:placementId/agent/:sessionThreadId`;
- `/jobs/:jobId` получает Job и перенаправляет в его workspace Jobs surface;
- `/job-runs/:jobRunId` получает run, затем Job и перенаправляет в nested run;
- `/jobs` открывает Jobs последнего workspace, а без него — Dashboard;
- `/nodes` открывает последнюю/первую Node, а при пустом inventory —
  `/nodes/pair`;
- `/projects/:projectId` остаётся ненавигационным compatibility/detail route
  для существующей reference model;
- `inspect` query и остальные безопасные query parameters сохраняются при
  redirect и переключении workspace tabs.

Generic `routeForRef(session)` может оставаться на resolver URL, потому что
сам `SessionRef` не содержит placement id. В местах, где placement известен,
нужно использовать прямой workspace-aware route builder.

## План реализации

### 1. Route model и workspace shell

Цель: сначала создать устойчивую вложенную структуру, не меняя внутренние
функции экранов.

- [ ] Добавить workspace-aware route builders для Agent, Workbench, Jobs,
  Job detail/run и выбранной session.
- [ ] Перестроить `App.tsx` на nested workspace routes с lazy boundaries на
  уровне трёх поверхностей.
- [ ] Создать `WorkspaceLayout`, который один раз разрешает placement и Node,
  показывает workspace header, path, status dimensions и tabs, затем рендерит
  child route через `Outlet`.
- [ ] Хранить последнюю workspace surface как локальную UI preference; default
  остаётся `agent`.
- [ ] Добавить resolver routes для старых session/job/run/node URLs.
- [ ] Валидировать контекст nested routes: session и Job обязаны принадлежать
  `placementId` из URL; mismatch даёт redirect к правильному workspace или
  явный not-found, но не смешивает данные двух workspace.
- [ ] Сохранить `WorkspaceDraftProvider` над всеми workspace routes, чтобы
  несохранённые editor drafts не терялись при переходе в Agent/Jobs и обратно.

Основные файлы:

- `apps/web/src/app/App.tsx`;
- новый `apps/web/src/features/workspaces/WorkspaceLayout.tsx`;
- новый `apps/web/src/features/workspaces/routes.ts`;
- `apps/web/src/workbench/references/refs.ts`;
- compatibility route components рядом с sessions/jobs/nodes.

Exit criteria:

- прямое открытие каждого canonical URL работает;
- legacy URLs приходят в тот же объект внутри workspace;
- browser Back/Forward восстанавливает surface и выбранный объект;
- query-encoded Inspector stack не теряется.

## Рекомендуемые implementation slices

1. **Routing foundation:** nested routes, route builders, ownership guards и
   compatibility redirects.
2. **Shell foundation:** collapsible Nodes sidebar, conditional Inspector и
   status primitives.
3. **Workspace Agent:** session list + существующая session surface.
4. **Workspace Workbench:** IDE grid, editor/diff modes, PTY и удаление Command
   block из UI.
5. **Workspace Jobs:** filtered list/create/detail/run.
6. **Overview surfaces:** Node Overview, Dashboard и Recent Activity.
7. **Regression/release:** unit, E2E, goldens, canonical docs и version bump.

Каждый slice должен оставлять `make l` зелёным. Version metadata меняется один
раз в финальном release slice, а не в промежуточных коммитах.

## Риски и меры

### Потеря существующей функциональности при визуальном переносе

Agent surface содержит больше, чем показано в прототипе: delayed messages,
evidence projection, agent projection и полный lifecycle. Они явно входят в
acceptance criteria и не должны исчезнуть ради визуального сходства.

### Async redirect loops и неверный workspace context

Legacy resolvers зависят от inventory/Job query. Redirect выполняется только
после успешного resolve, использует `replace` и проверяет ownership. Loading,
not-found и API failure должны быть отдельными состояниями.

### Monaco/xterm resize после изменения shell

Скрытие sidebar и открытие Inspector меняют размер pane без route change.
Workbench должен полагаться на container resize observation, а E2E проверяет
видимость editor/terminal и отсутствие overflow после обоих toggle.

### Смешение статусов

Нельзя выводить все backend enums одним универсальным Badge. Presentation
mapping обязан сохранять dimension, доступный label и независимость attention
от lifecycle.

### Ложная event log семантика

Recent Activity не называется raw log и не утверждает transition, которого нет
в данных. Расширение до node/workspace/check events откладывается до отдельного
Core projection в causality/trace slice.

### Дублирующий polling и streams

Workspace layout не должен сам запускать session SSE или Jobs polling. SSE
живет только в выбранной Agent session, Jobs polling — только в активной Jobs
surface, inventory остаётся общим shell query.

## Вне scope 0.2.6

- новый Core activity/event-log endpoint и raw event log screen;
- causality/trace UX item `9` и новые exact evidence contracts;
- изменение Rust protocol v2 или database migration;
- удаление bounded workspace command API из Core/Node;
- production permission matrix сверх уже существующих command guards;
- полный mobile navigation redesign;
- resizable/dockable panes, LSP, debugger и full IDE semantics;
- новый visual polish вне решений, необходимых для рабочего layout;
- изменение Job scheduler/runtime semantics.

## Completion criteria

`0.2.6` считается завершённой, когда одновременно выполнено следующее:

1. Sidebar является единственным списком Nodes/Workspaces, скрывается независимо
   от Inspector и не содержит sessions.
2. Клик по Node открывает Node Overview; отдельный global Nodes list отсутствует.
3. Workspace имеет три canonical surfaces: Agent, Workbench и Jobs.
4. Agent содержит выбор sessions и весь работающий session lifecycle 0.2.5.
5. Workbench одновременно даёт files/editor-or-diff/terminal и не показывает
   отдельный Command block.
6. Jobs фильтруются и создаются строго в текущем workspace; detail/run deep links
   работают.
7. Dashboard содержит четыре согласованные metrics и честный Recent Activity.
8. Context Inspector не занимает ширину без reference и сохраняет stack/deep
   links.
9. Legacy session/job/run/node URLs не ломаются.
10. Existing security, workspace boundary, provider admission, scheduled
    messages, Job scheduler и Core/Node behavior не изменены.
11. Unit/component/E2E/golden проверки обновлены, lazy renderer loading сохранён,
    `make c` проходит.
12. Canonical docs и вся implementation version metadata фиксируют `0.2.6`.
