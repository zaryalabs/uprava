Каноническое направление:
[A-008 Go to Source and Causality UX](../systems/areas/008-go-to-source-and-causality-ux.md).

## Цель среза

Закрыть пункт `9` feature queue первым полезным, но честным causality UX:
пользователь видит компактные шаги, может перейти к source, evidence, cause,
result или raw event и явно запросить bounded Deduction. Система не выдаёт
coarse provider activity или snapshot diff за точную причинность.

## API baseline

```text
GET  /api/v1/events
GET  /api/v1/events/{event_id}
POST /api/v1/references/resolve
GET  /api/v1/sessions/{session_thread_id}/trace
POST /api/v1/sessions/{session_thread_id}/deductions
GET  /api/v1/deductions/{deduction_id}
POST /api/v1/deductions/{deduction_id}/cancel
POST /api/v1/deductions/{deduction_id}/persist
```

## Инварианты

1. Event log остаётся source of truth; trace и narrative являются read
   projection/index поверх него.
2. `observed` Deduction step обязан иметь support refs из конкретного evidence
   snapshot; provider не может вернуть произвольную ссылку.
3. Deduction не продолжает provider session, не меняет workspace и не
   добавляется в live transcript/resume state.
4. Raw fallback ограничен по размеру и остаётся доступным при невалидном rich
   result.
5. Missing/offline/redacted/unsupported refs не маскируются fake resolution.
6. Snapshot diff и grouped provider activity остаются `coarse`; exact
   per-edit provenance отложен до отдельного event-quality slice.
