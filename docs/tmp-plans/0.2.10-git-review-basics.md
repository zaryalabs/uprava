# План реализации 0.2.10: Git and review basics

Статус: `implemented`

Целевая версия: `0.2.10`

Каноническое направление:
[A-010 Project Workspace Surface](../systems/areas/010-project-workspace-surface.md).

## Цель среза

Закрыть пункт `10` feature queue первым read-only review workflow: пользователь
видит типизированный branch/worktree snapshot, список staged, unstaged,
untracked и conflicted файлов, открывает bounded per-file diff и запускает
traceable checks с долговечной историей результатов.

## Scope

- [x] Типизированный Git snapshot в protocol, Node, Core и Web.
- [x] Сохранение последнего Git snapshot для Placement.
- [x] Changed-file list и diff scopes `all`, `staged`, `unstaged`.
- [x] Monaco diff editor с raw fallback для binary/truncated content.
- [x] Review panel с Git risks и snapshot freshness.
- [x] Async `make l` / `make c` entry points и check result history.
- [x] Causality refs для diff snapshots, hunks и checks.
- [x] Документация, release metadata и version bump `0.2.10`.

## Инварианты

1. Node Daemon остаётся authority для Git и filesystem facts; Browser и Core
   не читают node-local repository напрямую.
2. Срез не изменяет Git state: нет fetch, checkout, stage, commit, reset,
   discard или worktree creation.
3. Git вызывается без shell, с machine-readable output и bounded stdout/stderr.
4. Relative path проходит workspace boundary validation и передаётся после
   `--`; credentials не попадают в repo identity или UI.
5. Ahead/behind отражают только локально известный upstream и всегда имеют
   snapshot timestamp.
6. Snapshot diff остаётся coarse evidence. Он не выдаётся за exact provenance
   конкретного edit.
7. Check является bounded command с `intent=check`, durable result и
   `CheckResult` reference; interactive PTY не дублирует этот workflow.

## Вне scope

- GitHub/GitLab, PR/MR и CI provider integration;
- review comments и approval workflow;
- mutating Git actions;
- repository-specific check discovery или Tool Registry policy;
- remote network refresh.
