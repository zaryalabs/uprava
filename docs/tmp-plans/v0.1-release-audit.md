# Cortex 0.1 Release Audit

Status: complete.

Last updated: 2026-06-17.

This audit maps `v0.1-implementation-plan.md` to current evidence. It is a
working handoff document, not a release tag approval.

## Current Verification Evidence

- `make c` passed on 2026-06-17 after the Codex runtime protocol decision and
  audit updates.
- `cargo test --workspace -- --list` shows 41 Node tests, 8 protocol tests and
  76 Core tests covering protocol round trips, persistence, command dispatch,
  event ordering, session lifecycle, migration, security and provider adapter
  behavior.
- `npm --prefix apps/web run test -- --run --reporter=verbose` passed 13 files
  and 30 web tests covering routes, projection helpers, command registry,
  lifecycle controls, inventory state, block fallback and refs.
- `make web-e2e` passed the default browser suite after rerunning with local
  server permissions on 2026-06-17: shell plus deterministic warning/degraded
  structured-block assertions. The live-profile test is skipped unless
  `CORTEX_E2E_REAL_API=1`.
- Source-built live fake-provider browser E2E passed with
  `CORTEX_E2E_REAL_API=1` against host Core, Node and Web. That path now starts
  a session, renders it, clicks detach/attach, stop/resume, sends a post-resume
  browser turn and verifies reload reconstruction.
- Source-built fake-provider lifecycle smoke passed through
  `scripts/compose-smoke.sh` with Docker startup/restart checks disabled. It
  covered start, turn send, detach/attach, detached-turn rejection, approval
  resolve, interrupt from blocked runtime, stop/resume, post-resume turn and
  provider-error projection.
- Fresh current-source `make compose-smoke` passed from a clean Compose reset
  on 2026-06-17. The run rebuilt Core, Node and Web images, started the
  documented Core/Web/synthetic-Node profile, verified session start, turns,
  detach/attach, approval resolution, interrupt, stop/resume, post-resume turn,
  Core restart reconstruction, Node restart/reconnect and provider-error
  projection.
- `make codex-smoke` passed on 2026-06-17 after rerunning with local-server
  permissions against current source using the host Codex CLI, temporary host
  Core/Web/Node and the live Playwright profile.
- `playwright-cli` operator inspection passed on 2026-06-17 against a
  source-built local profile. The run covered trusted banner visibility,
  inventory/workspace/session navigation, session start, turn send, ordered
  runtime events, detach/attach, stop/resume, post-resume turn send, reload
  reconstruction, Inspector reference opening and deterministic error/degraded
  UI state. See `docs/tmp-plans/playwright-operator-inspection.md`.
- Codex runtime protocol spike completed on 2026-06-17. Local CLI evidence shows
  stable `codex exec` and `codex exec resume` JSON/non-interactive paths, while
  app-server is still experimental and requires a larger live-process owner
  subsystem. V0.1 therefore uses exec/resume continuity behind the Provider
  Adapter boundary. See `docs/tmp-plans/codex-runtime-protocol-spike.md`.
- `git diff --check` passed after the current changes.

Docker note: earlier retries failed before application startup while Docker Hub
metadata requests returned EOF for base images. A later retry on 2026-06-17
succeeded after metadata resolution became available and the local BuildKit
cache was warm.

## Release Contract Audit

1. Start Core Backend and Web Control Panel locally through Docker Compose.
   Status: proven.
   Evidence: `compose.yaml`, Dockerfiles and `make compose-smoke` wrapper exist;
   fresh current-source `make compose-smoke` rebuilt Core, Node and Web images,
   started the Compose profile on expected ports and completed the smoke flow.

2. Start at least one Node Daemon and enroll it in Core.
   Status: proven.
   Evidence: Node enrollment and heartbeat tests; source-built live fake and
   Codex smokes both enrolled a temporary Node with auto-approval.

3. See node status, heartbeat age, capabilities, degraded state and active
   sessions.
   Status: proven.
   Evidence: inventory read-model tests, `InventoryTreeContent` web test,
   `make web-e2e` deterministic stale/degraded rendering and live inventory
   assertions in the real-profile E2E. The recorded `playwright-cli` operator
   inspection showed reachable idle/active Node state, provider capabilities,
   same-workspace warning and deterministic degraded/error runtime state.

4. Register or open a project/workspace binding on the selected node.
   Status: proven.
   Evidence: placement validation tests, Node workspace validation tests,
   source-built live fake/Codex smokes selecting validated placements, and
   placement UI route tests.

5. Start a Core-managed Codex-backed session for that workspace, with persisted
   session state and provider resume continuity through the V0.1 exec/resume
   adapter.
   Status: proven.
   Evidence: `make codex-smoke` starts a Codex-backed session and completes a
   turn through Core/Web/Node. Node and Core tests cover provider resume refs,
   `codex exec resume`, bounded transcript fallback and persisted resume
   references. Known limits document that provider-native live process
   ownership, streaming output and real interrupt escalation are post-V0.1 work.

6. Send chat turns through Core-routed commands.
   Status: proven.
   Evidence: Core `send_turn_*` tests, Node fake/Codex send-turn tests,
   lifecycle smoke, live fake browser E2E, Codex smoke and the recorded
   `playwright-cli` operator inspection.

7. Receive ordered provider/runtime events in the session view.
   Status: proven.
   Evidence: event append monotonicity and gap tests, Node runtime sequence
   tests, web projection tests, live fake/Codex E2E render assertions and the
   operator inspection showing ordered runtime/turn/provider events after
   reload.

8. Detach from and reattach to the same session.
   Status: proven.
   Evidence: Core detach/attach tests, command registry/lifecycle control tests,
   expanded lifecycle smoke, live fake browser E2E and recorded operator
   inspection of disabled/enabled controls.

9. Interrupt, stop and resume where the provider adapter supports it.
   Status: proven for the V0.1 provider contract.
   Evidence: fake lifecycle smoke and browser E2E cover interrupt, stop and
   resume. The recorded operator inspection also covers stop, resume,
   post-resume turn send and resume after a deterministic runtime error.
   Core/Node tests cover Codex resume and unsupported interrupt mapping. Codex
   real interrupt escalation is explicitly outside the V0.1 exec/resume adapter
   contract.

10. Reload the UI and reconstruct nodes, workspace bindings, sessions,
    messages, runtime state and recent event history from Core persistence.
    Status: proven.
    Evidence: Core restart and session-history tests, source-built live fake
    browser reload assertion, recorded `playwright-cli` reload reconstruction
    and fresh current-source Compose Core restart smoke.

11. See stale/offline/error/resource warning states as UI state, not chat text.
    Status: proven.
    Evidence: inventory and projection tests, mocked Playwright warning/degraded
    test, runtime error message projection tests, fake provider error smoke and
    recorded operator inspection of same-workspace warning plus
    degraded/error runtime state.

12. See explicit trusted/local/non-production warning where the UI exposes node
    pairing or remote control.
    Status: proven.
    Evidence: trusted banner component, app shell route tests, runbook and
    deployment profile defaults, plus recorded `playwright-cli` inspection of
    the `local_trusted` banner and trusted pairing copy.

## Release Gate Checklist Audit

- Release contract satisfied end to end: proven by tests, source-built smokes,
  Codex smoke, operator inspection and fresh current-source Compose smoke.
- Docker Compose local profile starts documented Core/Web/fake path: proven by
  fresh current-source `make compose-smoke`.
- Trusted/local/non-production profile visible and documented: proven.
- Trusted/dev security baseline: proven for V0.1 by loopback defaults, CORS
  rejection, redaction, pairing expiry and revoke tests. Production hardening is
  correctly documented out of scope.
- Persistent Core state survives restart: proven by SQLite reopen tests and
  fresh current-source Compose Core restart smoke.
- SQLite migrations repeatable from empty and previous dev snapshots: proven by
  migration tests.
- Node idle/active/degraded/stale/offline behavior visible and history
  preserved: proven by tests and UI state coverage.
- Session history, messages, runtime events and provider resume reference stored
  in Core: proven by Core persistence and provider resume tests.
- Command idempotency prevents duplicate turns during retry/reconnect: proven by
  Node duplicate command/outbox tests and Core event dedupe tests.
- Event ordering monotonic per runtime session: proven by event append and gap
  tests.
- Event envelope refs and explicit unknown causality: proven by protocol and
  web ref tests.
- Control channel reconnect retries unacked events without duplicating UI:
  proven by Node outbox/reconnect tests, lifecycle smoke coverage and fresh
  current-source Compose Node restart smoke.
- Codex-specific behavior contained inside provider adapter: proven by module
  boundaries, tests and the documented V0.1 exec/resume protocol decision.
- Web does not depend on direct Node access: proven by API client routing through
  Core and E2E usage.
- Web uses internal surfaces, blocks, references and command registry: proven by
  tests and implementation.
- `CortexRef` supports V01 plus reserved future refs with unavailable states:
  proven by protocol and web ref tests.
- Lightweight artifact tree and agent projection surfaces: proven by Core and
  web tests.
- Artifact tree and timeline blocks preserve refs/fallbacks: proven by tests.
- Runtime events, warnings, approvals and errors visible as structured UI:
  proven by web tests, fake-provider smoke and recorded operator inspection.
- Automated Playwright E2E and documented `playwright-cli` path: automated E2E
  proven; operator path documented and recorded in
  `docs/tmp-plans/playwright-operator-inspection.md`.
- User-safe API errors and correlation ids: proven by Core tests and web error
  tests.
- Structured logs cover distributed paths: proven by implementation and
  observed `make compose-smoke` logs for heartbeat, control channel, command,
  event batch and runtime lifecycle paths.
- Known limitations documented: proven in runbook.
- `make c` passes: proven.
