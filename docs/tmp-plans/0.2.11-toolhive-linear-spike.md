# Spike ToolHive и Linear MCP для 0.2.11

Статус: `partial; superseded-by-manual-compose-acceptance`

Дата проверки: `2026-07-19`

Эта запись содержит воспроизводимые команды временного compatibility spike.
Долговечные продуктовые контракты зафиксированы в
[`agent-tooling-contracts.md`](../development/agent-tooling-contracts.md).

## Проверенный baseline

- ToolHive CLI `0.40.0`;
- commit `505df835ed73790bb9be7db7944ec772dc136a0e`;
- macOS arm64 artifact;
- SHA-256
  `77dbd6f657fa2ad9676b284beab8630e11f36e9014045993c0b7e6db3cd62dbb`;
- upstream: `https://mcp.linear.app/mcp`;
- transport: Streamable HTTP;
- локальный proxy endpoint: `http://127.0.0.1:18766/mcp`.

Checksum сверялся с официальным release-файлом до распаковки. Установленный в
систему binary не требовался: spike выполнялся из временного каталога, а
конфигурация и runtime state направлялись в `/private/tmp` через `XDG_*`.

## Exact commands

Проверка binary:

```sh
/private/tmp/uprava-toolhive-spike-0.40.0/thv version
```

Создание remote workload:

```sh
/private/tmp/uprava-toolhive-spike-0.40.0/thv run \
  https://mcp.linear.app/mcp \
  --name uprava-linear-spike \
  --remote-auth \
  --remote-auth-skip-browser \
  --remote-auth-timeout 5m \
  --remote-auth-callback-port 18765 \
  --proxy-port 18766
```

Диагностика без credential material:

```sh
/private/tmp/uprava-toolhive-spike-0.40.0/thv list --format json
```

Очистка:

```sh
/private/tmp/uprava-toolhive-spike-0.40.0/thv stop uprava-linear-spike
/private/tmp/uprava-toolhive-spike-0.40.0/thv rm uprava-linear-spike
```

Конкретные значения `XDG_CONFIG_HOME`, `XDG_DATA_HOME`, `XDG_CACHE_HOME` и
`XDG_RUNTIME_DIR` являются локальной деталью тестового запуска и должны
указывать на закрытый временный каталог.

## Что доказано

ToolHive:

1. принял remote MCP URL через `thv run URL`;
2. поднял локальный Streamable HTTP proxy;
3. обнаружил Bearer challenge и protected-resource metadata;
4. обнаружил authorization server `https://mcp.linear.app` и OAuth endpoints;
5. зарегистрировал OAuth client с PKCE;
6. дошёл до Linear consent screen;
7. показал безопасный workload status без access/refresh tokens;
8. корректно удалил workload; финальный `thv list --format json` вернул `[]`.

Standalone `thv proxy --target-uri ...` не подходит: в `0.40.0` эта команда
отклоняет target URI с path `/mcp`. Для remote server нужен именно
`thv run https://mcp.linear.app/mcp`.

При отсутствии secret manager ToolHive явно предупреждает, что OAuth tokens
не будут сохранены. Production adapter обязан настраивать approved local secret
store; Core получает только opaque credential reference и безопасное состояние
авторизации.

## Результат acceptance

Запись подтверждает compatibility discovery и достижение consent screen, но не
закрывает реальный product E2E. Утверждение о завершённых callback,
`tools/list`, read-only call и disconnect/reconnect было преждевременным.
Актуальная отдельная Compose topology и ручной checklist находятся в
[`agent-tooling-toolhive-linear.md`](../runbooks/agent-tooling-toolhive-linear.md).

Core по-прежнему не выводит подтверждение upstream revoke из одного локального
удаления workload: API возвращает `remote_revocation_confirmed = false`, пока
runtime не получил отдельное машинно проверяемое доказательство от upstream.

## Cleanup и данные

- ToolHive workload удалён;
- локальный proxy остановлен;
- системная установка не выполнялась;
- OAuth URL, state, verifier, tokens и credentials в документ не записаны;
- Read-only acceptance не изменял test workspace Linear.
