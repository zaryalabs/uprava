# Документация Uprava

`docs/` — единое каноническое дерево документации Uprava. Основной язык
долговечных продуктовых, архитектурных и процессных документов — русский.

Временные рабочие разделы [`polish/`](polish/) и
[`tmp-plans/`](tmp-plans/) могут оставаться на английском, если так удобнее для
текущей работы. Долговечные решения из них нужно переносить в основное русское
дерево.

Текущие разделы и документы:

- [`vision.md`](vision.md) — действующая vision продукта.
- [`product/`](product/) — эволюция продукта, очередь реализации, исторические
  стадии и инвентарь фич.
- [`systems/`](systems/) — общая архитектура и отдельные системные направления.
- [`development/`](development/) — технический стек, исходные заметки и
  self-hosting workflow.
- [`deploy/`](deploy/) — deployment, CI/CD и observability.
- [`versioning.md`](versioning.md) — правила SemVer и текущий release baseline.
- [`releases.md`](releases.md) — ledger выпущенных implementation slices.
- [`runbooks/`](runbooks/) — операционные инструкции.
- [`audit/audit-2026-07-09.md`](audit/audit-2026-07-09.md) — единый аудит
  архитектуры, качества кода, reliability, security и delivery для baseline
  `0.1.8`.

Временные реализационные планы живут в [`tmp-plans/`](tmp-plans/). Это рабочие
документы для активных промежуточных срезов разработки, а не канонический
источник продуктовых и архитектурных решений.
