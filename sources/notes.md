## AgentOps

**Как выглядит state-of-the-art AgentOps в 2026:**
Типичный production stack сегодня выглядит примерно так:

```
User / Event
      ↓
Workflow Orchestrator
(Temporal / LangGraph / Durable Execution Layer)
      ↓
Planner Agent
      ↓
Specialized Agents
(coding / support / retrieval / browser / finance)
      ↓
Tools & MCP servers
      ↓
State Store + Event Log
(Postgres / Redis / S3 / Vector DB)
      ↓
Observability Layer
(LangSmith / Langfuse / OpenTelemetry / Phoenix)
```

Ключевой момент: агент больше не “процесс”.  
Агент — это _event-driven state machine_.

**При этом SOTA подход это stateless agent и sandbox:**

1. Workflow worker получает тикет
2. Создаёт workspace
3. Клонирует repo в sandbox
4. Агент читает файлы через tools
5. Агент редактирует файлы
6. Запускает tests через bash tool
7. Создаёт PR через GitHub API tool
8. Workflow засыпает
9. GitHub webhook будит workflow
10. Агент проверяет CI
11. Обновляет Jira

Важно: **долгоживущим является workflow state, а не контейнер**. Контейнер с bash может жить 30 секунд, 20 минут или быть пересоздан. Durable runtime хранит “где мы остановились”. Например Temporal, хранит историю workflow и переисполняет/восстанавливает состояние через durable execution [source](https://github.com/temporalio/temporal?utm_source=chatgpt.com).
